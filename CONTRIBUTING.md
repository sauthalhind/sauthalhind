# Contributing

## Before you change anything

- Read `AI_CONTEXT.md`
- Read `PROJECT_STRUCTURE.md`
- Check the relevant app route or component

## Workflow

1. Make a focused change
2. Run `npm run build`
3. Test the affected page in the browser
4. Commit the change
5. Push to GitHub

## Standards

- Keep RTL and Arabic copy intact
- Preserve mobile-first design
- Avoid breaking existing routes
- Keep secrets out of the repo

## Good pull request changes

- Bug fix
- Feature add
- Design refinement
- API update
- Documentation update
